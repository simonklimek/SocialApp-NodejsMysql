Security, privacy and legal issues:

In future development I scheduled to implement security features listed below:

-404 detections: 404 detection assumes that a user who hits a lot of 404 errors in a short period of time is scanning for something (presumably a vulnerability) and locks them out accordingly.

-File Change Detection: script / plugin will look only at files to the last check instead of comparing them with a remote Github thereby taking into account whether or not you modify the files yourself.

-File Permission: keeps certain file / users /groups permissions in the project structure in the key areas of the site.

-Enable Password Salts: A secret key makes your site harder to hack and access by adding random elements to the password.

-Edit URI Header: Removes the RSD (Really Simple Discovery) header.

 

-Rules to enable against:

XXE: External Entity Expansion

SQL Injection vulnerability

XSS: Cross Site Scripting

XSS HTML injection in inline JavaScript

Malicious File Upload

LFI: Local File Inclusion
LTI: Directory Traversal

Local Brute Force Attack

HTML5 DDoS – I wrote article about it in 2014, link: https://simonsnetwork.co.uk/html5-vs-ddos/

 

 

Privacy and legal issues storing the data:

EU users must agree to the use of cookies or local browser storage to protect users’ privacy.

In Social Club app authentication for login doesn’t require user consent to create session in the browser storage.

Customer should agree to not upload illegal, explicit post pictures and avatars, and the post content must not be related with illegal topics.

Customer should agree to the data which he / she provides is valid and it belongs to its customer.

Customer should also agree to the risks related with storing its data in MySql database protected with root password.
